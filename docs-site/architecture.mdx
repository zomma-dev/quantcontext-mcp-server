## The LLM Boundary

QuantContext enforces a strict boundary between what LLMs do and what code does:

```
┌─────────────────────────────────────────────────────────┐
│                       LLM Layer                          │
│  Parses user intent ("find cheap stocks")                │
│  Selects the right tool (screen_stocks)                  │
│  Chooses parameters ({pe_lt: 15})                        │
│  Explains results in natural language                    │
│  Decides what to compute next                            │
│                                                          │
│  The LLM NEVER computes a number.                        │
│  The LLM NEVER touches market data.                      │
│  The LLM NEVER estimates a Sharpe ratio.                 │
└──────────────────────┬──────────────────────────────────┘
                       │  MCP Protocol (JSON-RPC over stdio)
┌──────────────────────▼──────────────────────────────────┐
│                  QuantContext Layer                       │
│  Receives structured tool calls                          │
│  Validates inputs                                        │
│  Executes deterministic Python computation               │
│  Returns structured JSON                                 │
│                                                          │
│  Every number is computed from data.                     │
│  Results are fully reproducible.                         │
│  No LLM calls in the computation path.                   │
└──────────────────────┬──────────────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────────────┐
│                     Data Layer                            │
│  yfinance: daily OHLCV, fundamentals                     │
│  Kenneth French Data Library: Fama-French factors        │
│  Local cache: ~/.quantcontext/cache/                     │
│  Bundled universe lists: S&P 500, Russell 2000, Nasdaq   │
└─────────────────────────────────────────────────────────┘
```

## Why Determinism Matters for Trading

LLMs hallucinate numbers. QuantContext ensures every number returned to an agent is:

1. **Computed from actual market data**: not generated, estimated, or hallucinated
2. **Deterministic**: same inputs produce identical outputs, always
3. **Auditable**: the computation path is pure Python with no randomness

## Data Sources and Caching

| Source | Data | Cache | Location |
|--------|------|-------|----------|
| yfinance | Daily OHLCV prices | Refreshed when stale | `~/.cache/quantcontext/prices.parquet` |
| yfinance | Fundamentals (PE, ROE, etc.) | 24 hours per ticker | `~/.cache/quantcontext/financials/` |
| Kenneth French | Fama-French + momentum factors | Cached indefinitely | `~/.cache/quantcontext/ff_factors.parquet` |
| Wikipedia | Universe constituent lists | Cached on first fetch | `~/.cache/quantcontext/sp500_tickers.json` |

**First call:** Downloads market data (~10s for S&P 500). Subsequent calls use the cache.

**Performance:** Screening under 1s, backtesting 3–8s.

**Total disk usage:** ~50MB for full S&P 500 coverage.

## Transport

QuantContext uses the MCP stdio transport. The server reads JSON-RPC messages from stdin and writes responses to stdout.

```bash
# Started automatically by the MCP client. You never run this manually.
quantcontext
```

## Use from Python

The tools are also importable directly — no MCP client or agent required. This is useful for existing Python scripts, notebooks, or automated pipelines.

```python
from quantcontext.server import screen_stocks, backtest_strategy, factor_analysis
import asyncio, json

# Screen for candidates
result = json.loads(asyncio.run(screen_stocks(
    universe="sp500",
    screen_type="fundamental_screen",
    config={"pe_lt": 15, "roe_gt": 12},
)))

# Backtest a strategy
bt = json.loads(asyncio.run(backtest_strategy(
    stages=[{"order": 1, "type": "screen", "skill": "fundamental_screen", "config": {"pe_lt": 15, "roe_gt": 12}}],
    universe="sp500",
    rebalance="monthly",
    start_date="2022-01-01",
)))
print(bt["metrics"])

# Pipe the equity curve straight into factor analysis
fa = json.loads(asyncio.run(factor_analysis(
    equity_curve=bt["full_equity_curve"]
)))
print(fa["alpha_annualized"], fa["alpha_tstat"])
```

All three functions are async and return JSON strings. Strategies must be expressed using the built-in screen types (`fundamental_screen`, `momentum_screen`, `quality_screen`, etc.) — arbitrary Python logic in the pipeline is not supported.
